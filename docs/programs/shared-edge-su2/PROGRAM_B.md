# Program B: Shared-Edge SU(2) Finite Bridge v0.1

Status: execution plan.

## Objective

Build the first finite, auditable shared-edge SU(2) kernel program that is strong enough to test nonseparability and disciplined enough not to collapse back into product-kernel tranche production.

The program is successful at v0.1 when it creates the minimum reproducible substrate for shared-edge computation:

- convention lock;
- inherited gate reproduction;
- falsification rule;
- catalog entry schema;
- product-kernel regression boundary;
- first shared-edge witness design;
- CI-ready implementation issues.

## Non-objectives

This program does not:

- continue the old tranche sequence;
- produce new Program A spectra;
- update Clay-route percentages;
- expand the fixed no-claim statement;
- claim a continuum Yang-Mills mass gap;
- accept unexplained numerical coincidences;
- treat a nonzero shared-edge coefficient as success unless nonseparability is demonstrated.

## Core lemma imported as Program A regression

Program A is summarized by a regression theorem, not by a live research queue:

Under a Wilson kernel built from class-function factors on independent plaquette variables with a single central Haar projection, the transfer operator on the scalar-coupled invariant subspace acts as a tensor product of one-plaquette channel matrices, normalized by the topology factor `1 / A^0_00`, with cross-l-triple entries killed by Schur orthogonality.

Program B must break, modify, or bypass at least one of the theorem's hypotheses:

1. independent plaquette variables;
2. class-function-only factors;
3. single central Haar projection;
4. scalar-only coupling with product-separable channel matrices.

## Shared-edge hypothesis

A genuine shared-edge kernel must create a coefficient tensor whose observable structure cannot be reconstructed as a product of independent one-plaquette channel matrices after any admissible basis change declared in `CONVENTIONS.md`.

This is the key difference between Program B and Program A. Program A discovered a product theorem late. Program B begins by defining how it would detect failure to escape that theorem.

## Sequence

### Phase 0 — Program skeleton

- Create `README.md`, `CONVENTIONS.md`, `GATES.md`, `FALSIFICATION.md`, `CATALOG.md`, and issue queue.
- Open a PR as the visible enforcement point.
- No new kernel values.

### Phase 1 — Gate reproduction

- Reproduce the U(1) null in the declared convention.
- Reproduce Phi_111 through a Wigner R-table path in the declared convention.
- Record exact commands, expected values, tolerances, and artifacts.

### Phase 2 — Product-kernel regression harness

- Encode the Program A product theorem as tests.
- Include K3 face regression and orbit/product structure only as frozen regressions.
- Fail CI if a later change mutates Program A semantics without a deliberate version bump.

### Phase 3 — Shared-edge minimal model

- Define the smallest shared-edge graph that makes an independence violation visible.
- Compute a predeclared coefficient or observable.
- Run the separability falsification test before any interpretive claim.

### Phase 4 — Catalog expansion

- Convert every accepted term/value into a catalog entry.
- Require cross-references to conventions, gates, code, tests, and status.
- Keep the catalog non-final and self-branching while keeping values reproducible.

## Definition of done for v0.1

Program B v0.1 is done when:

- a PR contains the program skeleton;
- the issue queue captures implementation tasks;
- inherited gates are specified with exact reproduction requirements;
- falsifiability is explicit;
- no new kernel-scope computation has been performed before conventions and gates are declared.
