# Shared-Edge SU(2) Finite Bridge Program v0.1

Status: program seed. This is the start of the next phase, not a formalization of Part One and not another tranche in the previous sequence.

## Purpose

This program moves from the completed product-kernel arc into a new finite SU(2) design space where shared-edge coupling is first-class. The point is not to preserve the old proof narrative. The point is to learn from it, preserve the usable machinery, and then build a falsifiable kernel program whose first result is not a theorem claim but an auditable operating system for computation.

Program A is now treated as a source archive: prior values, product-kernel regressions, convention seams, and interpretive gates. Program B is the new branch: shared-edge kernels, nonseparability witnesses, explicit convention locking, and a Synergetics-inspired catalog that ties terms to values, commands, tests, and cross-references.

## Public claim boundary

This is a finite-cutoff computational and representation-theoretic program. It does not address the continuum Yang-Mills existence and mass-gap problem, which requires uniformity in cutoff, volume, and coupling that this program does not establish.

## What changes now

We do not continue by adding one more tranche. We start with a new procedural architecture:

1. declare conventions before computation;
2. declare gates before accepting inherited values;
3. declare falsification before claiming nonseparability;
4. catalog terms as operational entries, not prose glosses;
5. keep values reproducible by command, fixture, or proof fragment;
6. distinguish product-kernel regression from shared-edge discovery.

## Workstreams

### 1. Kernel design

Define a shared-edge SU(2) kernel that breaks the independence assumption of the product-kernel theorem in a controlled way. The initial kernel must be small enough to audit and rich enough to test whether a coefficient remains secretly product-separable.

### 2. Convention lock

Canonicalize Wigner phase, representation labels, Haar normalization, character/class-function inner products, Wilson coefficient normalization, scalar-coupled basis order, topology scalar, and matrix ordering before any new values are accepted.

### 3. Gate reproduction

Inherited gates are not trusted until reproduced in the declared convention: the U(1) null and Phi_111 Wigner reproduction are interpretive gates, not cosmetic checks.

### 4. Falsification-first computation

Before computing a new shared-edge coefficient, define what outcome would mean the shared-edge design failed to escape product-kernel structure.

### 5. Catalog construction

Build a catalog of terms and values using entries with fields for definition, role, source, cross-reference, reproduction command, status, and falsification implication.

## First stable state

The first stable state is a public, auditable skeleton with conventions, gates, falsification rule, catalog schema, and issue-backed implementation work. It is not a mass-gap claim, not a continuum theorem, and not a monograph.
