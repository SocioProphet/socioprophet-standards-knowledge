# PARA projection rules v0.1

## Purpose

PARA is a human-facing projection over the canonical object model.
It is not the canonical ontology.

## Mapping

### Projects
Active bounded efforts with a defined outcome.

### Areas
Durable responsibility surfaces.

### Resources
Reusable reference and asset collections.

### Archives
Historical, superseded, or replayable retained material.

## Rule

A descriptor MAY project into PARA containers, but PARA assignments MUST NOT replace canonical fields such as domain, category, task, policy, provenance, or stewardship.

## Consequence

Search, recommendation, and governed workflows should use canonical descriptor fields first and PARA only as a human organization aid.
